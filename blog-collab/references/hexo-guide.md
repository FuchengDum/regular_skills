# Hexo 技术博客写作规范

## Front-matter 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| title | 是 | 文章标题 |
| date | 是 | 发布日期，格式 YYYY-MM-DD HH:mm:ss |
| tags | 否 | 标签列表 |
| categories | 否 | 分类列表 |
| updated | 否 | 更新日期 |
| comments | 否 | 是否开启评论，默认 true |
| permalink | 否 | 自定义永久链接 |

## 技术文章结构模板

### 教程类文章

```
前言（为什么写这篇）
↓
环境准备（版本要求）
↓
核心概念（必要的背景知识）
↓
实现步骤（分步骤，每步有代码）
↓
完整示例（可运行的完整代码）
↓
常见问题（FAQ）
↓
总结与延伸
```

### 问题解决类文章

```
问题描述（现象、错误信息）
↓
原因分析（为什么会出现）
↓
解决方案（具体步骤）
↓
验证方法（如何确认已解决）
↓
预防措施（如何避免再次发生）
```

### 技术对比类文章

```
背景介绍（为什么要对比）
↓
对比维度（性能、易用性、生态等）
↓
详细对比（表格 + 说明）
↓
使用场景建议
↓
结论
```

## 代码示例规范

### 代码块格式

````markdown
```javascript
// 添加注释说明关键步骤
const example = () => {
  // 实现逻辑
};
```
````

### 代码注释要求

- 关键逻辑必须有注释
- 避免显而易见的注释
- 复杂算法需要说明思路

### 代码完整性

- 提供可直接运行的完整示例
- 标注依赖版本
- 说明运行环境

## Markdown 格式规范

### 标题层级

- H1: 仅用于文章标题（由 front-matter title 生成）
- H2: 主要章节
- H3: 子章节
- H4: 细分内容（尽量避免）

### 列表使用

- 无序列表：并列项目
- 有序列表：有顺序的步骤
- 任务列表：检查清单

### 强调格式

- **粗体**：重要概念、关键词
- *斜体*：术语首次出现
- `行内代码`：代码、命令、文件名

## SEO 优化建议

- 标题包含核心关键词
- 前言包含文章摘要
- 使用有意义的标题层级
- 图片添加 alt 描述

## 图片使用规范

### 图片存放位置

推荐使用文章专属目录：
```
source/_posts/
  ├── article-name.md
  └── article-name/
      ├── cover.jpg
      └── section-1.jpg
```

### 图片引用格式

```markdown
![图片描述](<image.jpg>)
```

注意：
- 使用相对路径
- 文件名包含空格时使用 `<>` 包裹
- 必须提供有意义的 alt 文本

### 图片命名规范

- 封面图：`cover.jpg`
- 章节图：`section-{章节名}.jpg` 或 `{序号}-{描述}.jpg`
- 使用小写字母和连字符
- 避免中文文件名

### 图片优化建议

- 格式：JPG（照片/渐变）、PNG（图表/透明背景）
- 大小：< 500KB
- 尺寸：宽度 800-1200px
