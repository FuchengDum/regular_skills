# 技术对比类文章模板

## 适用场景

- 框架/库选型对比
- 技术方案对比
- 工具比较
- 语言/生态对比

## 核心特点

- **数据驱动**：有数据支撑的结论
- **表格对比**：清晰直观的对比表
- **中立客观**：不偏向任何一方

## 标准结构

```
## 前言
- 对比背景（为什么要做这个对比）
- 对比对象介绍
- 评估维度说明

## 对比对象概述
### [对象A]
- 简介
- 核心特点
- 适用场景

### [对象B]
...

## 对比维度
- 维度选择依据
- 评分标准说明

## 详细对比

### 维度一：[性能]
| 指标 | 对象A | 对象B | 说明 |
|------|-------|-------|------|
| 指标1 | 数据 | 数据 | 解释 |
| 指标2 | 数据 | 数据 | 解释 |

**分析**：[对比分析]

### 维度二：[易用性]
...

### 维度三：[生态系统]
...

## 综合评分

| 维度 | 对象A | 对象B | 权重 |
|------|-------|-------|------|
| 性能 | 8/10 | 7/10 | 30% |
| 易用性 | 7/10 | 9/10 | 25% |
| ... | ... | ... | ... |
| **总分** | **X** | **Y** | 100% |

## 场景建议
- 场景A 选择 [对象X]，因为...
- 场景B 选择 [对象Y]，因为...
- 场景C 考虑混合使用...

## 结论
- 总结对比结果
- 给出明确建议
- 强调选择取决于具体需求
```

## Claude 架构设计提示词

```
请为以下技术对比类文章设计架构：

对比对象：{objects}（如 React vs Vue）
对比目的：{purpose}
目标读者：{audience}

请使用 CoT 思维链分析：

1. 【维度选择】应该从哪些维度对比？依据是什么？
2. 【数据来源】每个维度的数据从哪里获取？
3. 【权重分配】不同场景下各维度权重如何？
4. 【客观性检查】如何避免主观偏见？

输出架构：
- 文章大纲
- 对比维度列表及权重
- 需要的数据/基准测试
- 表格结构设计
```

## Codex 内容填充提示词

```
请根据以下架构填充技术对比类文章内容：

架构：
{architecture}

要求：
1. 每个对比维度有数据支撑
2. 使用 Markdown 表格展示对比
3. 保持客观中立，优缺点都要说
4. 场景建议要具体实用
5. 避免绝对化结论
```

## Gemini 审核重点

- [ ] 对比数据是否准确、是否过时
- [ ] 是否有明显偏向某一方
- [ ] 是否遗漏重要对比维度
- [ ] 版本是否是最新稳定版
- [ ] 场景建议是否合理
- [ ] 是否需要补充基准测试数据
