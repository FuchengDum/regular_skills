# 深度解析类文章模板

## 适用场景

- 底层原理解析
- 源码分析
- 架构设计剖析
- 技术演进分析

## 核心特点

- **深入原理**：不停留在 API 层面
- **图文结合**：流程图、架构图辅助理解
- **理论深度**：有学术/工程理论支撑

## 标准结构

```
## 引言
- 问题引入（为什么要深入了解这个）
- 文章目标
- 前置知识要求

## 背景知识
- 相关概念回顾
- 历史演进（可选）
- 设计动机

## 底层原理
### 核心概念
- 概念定义
- 概念关系图

### 工作机制
- 流程图/时序图
- 分步骤说明
- 关键点解释

## 源码分析（可选）
### 核心模块
- 模块职责
- 关键代码片段（带行号）
- 代码注释解析

### 关键函数
```language
// 源码位置: path/to/file.js:123
function keyFunction() {
  // 注释：解释这行做什么
  ...
}
```

## 实践应用
- 原理如何指导实践
- 代码示例
- 性能优化建议

## 延伸思考
- 设计取舍分析
- 与其他方案对比
- 未来演进方向

## 总结
- 核心要点回顾
- 思维导图（可选）
- 推荐深入阅读
```

## Claude 架构设计提示词

```
请为以下深度解析类文章设计架构：

主题：{topic}
深度：{depth}（原理层/源码层/设计层）
目标读者：{audience}（需要中高级水平）

请使用 CoT 思维链分析：

1. 【问题引入】用什么问题引起读者兴趣？
2. 【知识图谱】涉及哪些概念？它们的关系是什么？
3. 【深度规划】从哪个层面切入？深入到什么程度？
4. 【图表规划】需要哪些图表辅助理解？
5. 【源码选择】（如果涉及）分析哪些关键源码？

输出架构：
- 大纲（章节 + 深度标记）
- 知识点依赖图
- 图表清单（类型、位置、描述）
- 源码分析清单（文件、函数、行号范围）
```

## Codex 内容填充提示词

```
请根据以下架构填充深度解析类文章内容：

架构：
{architecture}

要求：
1. 原理解释要准确，引用权威来源
2. 源码分析标注版本和文件路径
3. 复杂流程使用 Mermaid 图表
4. 概念解释后给出代码验证
5. 保持逻辑连贯，层层递进
```

## Gemini 审核重点

- [ ] 原理解释是否准确
- [ ] 源码版本是否是分析的版本
- [ ] 图表是否准确表达了概念关系
- [ ] 是否有过度简化导致的错误
- [ ] 引用的资料是否权威
- [ ] 是否遗漏重要的实现细节
