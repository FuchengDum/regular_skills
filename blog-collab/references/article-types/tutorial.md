# 教程类文章模板

## 适用场景

- 新技术/框架入门教程
- 功能实现指南
- 环境配置教程
- 工具使用指南

## 核心特点

- **渐进式教学**：从简单到复杂，循序渐进
- **可复制运行**：所有代码示例可直接复制运行
- **完整上下文**：包含环境、依赖、版本信息

## 标准结构

```
## 前言
- 文章目标
- 目标读者画像
- 预期收获

## 环境准备
- 运行环境要求
- 依赖版本说明（精确到小版本）
- 安装命令

## 核心概念
- 必要的背景知识
- 关键术语解释
- 概念关系图（可选）

## 分步实现
### 步骤 1: [标题]
- 说明
- 代码示例
- 预期结果

### 步骤 2: [标题]
...

## 完整示例
- 可运行的完整代码
- 运行命令
- 预期输出

## 常见问题 (FAQ)
- Q1: [问题]
  - A: [解答]
- Q2: [问题]
  - A: [解答]

## 总结与延伸
- 要点回顾
- 进阶资源链接
- 相关文章推荐
```

## Claude 架构设计提示词

```
请为以下教程类文章设计架构：

主题：{topic}
目标读者：{audience}（初学者/中级/高级）

请使用 CoT 思维链分析：

1. 【目标分析】读者学完能做什么？
2. 【知识点拆解】需要掌握哪些概念？依赖关系是什么？
3. 【难度曲线】如何安排从易到难的学习路径？
4. 【代码规划】每个步骤需要什么代码示例？

输出架构：
- 大纲（章节 + 要点）
- 每个章节的目标和预计篇幅
- 代码示例清单（位置、类型、语言、行数预估）
- 环境依赖清单
```

## Codex 内容填充提示词

```
请根据以下架构填充教程类文章内容：

架构：
{architecture}

要求：
1. 代码必须可直接运行，注明运行环境
2. 每个步骤代码前后有说明和预期输出
3. 使用渐进式示例（简单→完整）
4. 关键代码行添加注释
5. 包含常见错误的处理说明
```

## Gemini 审核重点

- [ ] 环境依赖版本是否最新稳定版
- [ ] 代码示例是否可直接运行
- [ ] 步骤是否有遗漏
- [ ] 是否适合目标读者水平
- [ ] 前置知识是否说明清楚
