# 问题解决类文章模板

## 适用场景

- 错误/异常解决方案
- 踩坑记录与解决
- 性能问题排查
- 兼容性问题处理

## 核心特点

- **关键词丰富**：便于搜索引擎收录
- **直接给出方案**：快速解决读者问题
- **场景完整**：包含原因分析和预防措施

## 标准结构

```
## 问题描述
- 现象描述（用户看到什么）
- 复现步骤
- 环境信息

## 错误信息
- 完整错误日志/截图
- 关键错误关键词（便于 SEO）

## 原因分析
- 根本原因
- 可能的多种原因列表
- 排查思路

## 解决方案
### 方案一：[推荐方案]
- 步骤说明
- 代码/命令
- 注意事项

### 方案二：[备选方案]
...

## 验证方法
- 如何确认问题已解决
- 验证命令/测试代码

## 预防措施
- 如何避免再次发生
- 最佳实践建议
- 相关配置推荐

## 相关问题
- 类似问题链接
- 延伸阅读
```

## Claude 架构设计提示词

```
请为以下问题解决类文章设计架构：

问题：{problem}
错误信息：{error_message}
环境：{environment}

请使用 CoT 思维链分析：

1. 【问题定位】这个错误的根本原因是什么？
2. 【多因分析】还有哪些可能导致相同错误？
3. 【方案设计】有哪些解决方案？优先级如何？
4. 【SEO 关键词】读者可能搜索哪些关键词？

输出架构：
- 文章标题（包含关键错误信息）
- 大纲结构
- 每个章节的要点
- 需要的代码/命令示例
```

## Codex 内容填充提示词

```
请根据以下架构填充问题解决类文章内容：

架构：
{architecture}

要求：
1. 错误信息使用代码块展示
2. 解决方案按推荐程度排序
3. 每个方案包含完整的命令/代码
4. 添加「适用场景」说明每个方案的使用条件
5. 验证步骤要具体可执行
```

## Gemini 审核重点

- [ ] 解决方案是否实际有效
- [ ] 是否有更简单/更好的方案
- [ ] 错误原因分析是否准确
- [ ] 相关依赖版本是否有变化
- [ ] 安全注意事项是否提及
