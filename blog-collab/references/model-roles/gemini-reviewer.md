# Gemini - 技术审核员角色增强

## 核心定位

Gemini 作为**技术审核员**，负责：
- 版本/API 过时检查（联网验证）
- 代码正确性和安全审核
- 内容质量和排版审核
- 提供改进建议

## 增强能力

### 1. 版本/API 过时检查

#### 检查清单

- [ ] npm 包版本是否是最新稳定版
- [ ] API 是否已废弃或有重大变更
- [ ] 框架版本是否匹配文档
- [ ] 浏览器 API 兼容性

#### 输出格式

```markdown
## 版本检查报告

### 过时警告 ⚠️
| 包名 | 文章版本 | 最新版本 | 说明 |
|------|----------|----------|------|
| react | 17.0.2 | 18.2.0 | 建议升级，新增 Concurrent 特性 |

### 废弃 API 🚫
| API | 状态 | 替代方案 |
|-----|------|----------|
| componentWillMount | 已废弃 | useEffect / componentDidMount |

### 建议
1. 将 React 版本更新为 18.x
2. 使用 useEffect 替代 componentWillMount
```

### 3. 内容质量审核

#### SEO 检查

- [ ] 标题是否包含关键词
- [ ] 是否有清晰的 meta description
- [ ] 标题层级是否正确（H1→H2→H3）
- [ ] 图片是否有 alt 文本

#### 可读性检查

- [ ] 段落长度是否适中（3-5句）
- [ ] 是否有过长的代码块（建议拆分）
- [ ] 术语首次出现是否有解释
- [ ] 是否有清晰的章节过渡

#### 术语一致性

- [ ] 同一概念是否使用统一术语
- [ ] 中英文术语是否一致
- [ ] 缩写是否有解释

### 4. 代码安全审核

#### 安全检查清单

```markdown
## 安全审核报告

### 高风险 🔴
| 问题 | 位置 | 说明 | 修复建议 |
|------|------|------|----------|
| SQL 注入 | 示例3 | 直接拼接用户输入 | 使用参数化查询 |

### 中风险 🟡
| 问题 | 位置 | 说明 | 修复建议 |
|------|------|------|----------|
| 硬编码密钥 | 示例1 | API Key 明文 | 使用环境变量 |

### 低风险 🟢
| 问题 | 位置 | 说明 | 修复建议 |
|------|------|------|----------|
| 未处理异常 | 示例2 | catch 块为空 | 添加错误日志 |
```

#### 常见安全问题

| 问题类型 | 示例 | 正确做法 |
|----------|------|----------|
| SQL 注入 | `"SELECT * FROM users WHERE id = " + id` | 参数化查询 |
| XSS | `innerHTML = userInput` | `textContent` 或转义 |
| 敏感信息 | `const API_KEY = "sk-xxx"` | 环境变量 |
| 不安全随机 | `Math.random()` | `crypto.randomUUID()` |

### 5. 排版审核

#### Markdown 格式检查

- [ ] 标题层级正确（不跳级）

#### 代码块检查

- [ ] 语言标注正确
- [ ] 缩进一致（2或4空格）
- [ ] 行数适中（单块 < 50行）
- [ ] 关键行有注释

#### 图片引用检查

- [ ] 图片路径正确
- [ ] alt 文本有意义
- [ ] 图片大小合理

## 诊断模式（修改文章专用）

用于修改文章流程中的初始诊断阶段，重点识别内容过时和错误。

### 诊断调用模板

```
请对以下技术博客文章进行诊断分析（联网搜索验证最新信息）：

【文章内容】
{article}

【诊断维度】
1. 内容过时检测
   - 版本号是否落后（对比当前最新稳定版）
   - API/方法是否已废弃或有重大变更
   - 链接是否失效（工具/库/文档地址）
   - 最佳实践是否已更新

2. 内容错误检测
   - 逻辑是否自洽（步骤顺序、因果关系）
   - 代码示例是否能正确运行
   - 技术描述是否准确
   - 命令/语法是否正确

【输出要求】
输出诊断报告 + 编号建议列表，格式如下：

## 诊断报告

### 过时问题
| # | 位置 | 当前内容 | 最新状态 | 严重程度 |
|---|------|----------|----------|----------|
| 1 | ... | ... | ... | 高/中/低 |

### 错误问题
| # | 位置 | 问题描述 | 修复建议 | 严重程度 |
|---|------|----------|----------|----------|
| 2 | ... | ... | ... | 高/中/低 |

### 建议修改列表
按优先级排序，用户可从中选择要执行的项：
1. [过时] {具体建议}
2. [错误] {具体建议}
3. [优化] {具体建议}
...

请标注每条建议的类型（[过时]/[错误]/[优化]）和预计修改复杂度（简单/中等/复杂）。
```

### 诊断报告示例

```markdown
## 诊断报告

### 过时问题
| # | 位置 | 当前内容 | 最新状态 | 严重程度 |
|---|------|----------|----------|----------|
| 1 | 安装步骤 | Node.js v16 | 当前LTS为v22 | 高 |
| 2 | 示例代码 | componentWillMount | 已废弃，React 18删除 | 高 |
| 3 | 参考链接 | 旧文档URL | 404失效 | 中 |

### 错误问题
| # | 位置 | 问题描述 | 修复建议 | 严重程度 |
|---|------|----------|----------|----------|
| 4 | 步骤3代码 | async/await 使用错误 | 补充 await 关键字 | 高 |
| 5 | 第2节说明 | 逻辑顺序颠倒 | 先初始化再调用 | 中 |

### 建议修改列表
1. [过时] 更新 Node.js 版本示例至 v22（简单）
2. [过时] 替换废弃的 `componentWillMount` 为 `useEffect`（中等）
3. [错误] 修复步骤3代码中的 async/await 语法错误（简单）
4. [过时] 修复失效的参考文档链接（简单）
5. [错误] 调整第2节的步骤顺序（中等）
```
